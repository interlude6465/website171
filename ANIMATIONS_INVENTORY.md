# Animations Inventory — myVicRoads v1.3.5

Generated by `decode_animations.py` (re-runnable). Source data is binary AXML in
`apk_extracted/res/{anim,anim-v21,animator,animator-v21,interpolator,interpolator-v21}/`,
decoded with `pyaxmlparser.AXMLPrinter`. AVD drawables were already decoded by
the earlier `decode_apk.py` pass; we copy the animation-flavoured ones into
`apk_loot/animations/avd/`.

The headline finding: **the APK ships almost zero app-specific animation XML.**
The 55 binary AXML files under `res/anim*/`, `res/animator*/`, and
`res/interpolator*/` are *all* AndroidX AppCompat + MaterialComponents
boilerplate (checkbox toggles, radio button morphs, FAB show/hide motion specs,
bottom-sheet slides, fragment open/close). The two genuine in-house animations
are the password-eye AVDs (`avd_show_password`, `avd_hide_password`).

The actual UI animation in the live app is driven from **Jetpack Compose** —
DEX scan finds `AnimatedVisibility`, `Crossfade`, `EnterTransition`,
`ExitTransition`, `AnimationSpec`, `CubicBezierEasing`, `keyframes`,
`SwipeRefreshLayout`, and an `Impulse` symbol. Concrete easing tuples observed
in error-string fingerprints (`0.2,0.0,0.8,1.0`, `0.4,0.0,1.0,1.0`,
`0.0,0.0,0.65,1.0`, `0.1,0.0,0.45,1.0`) are the Material 3 standard / emphasised
/ decelerated / accelerated curves — those become the basis for the CSS easing
in §7.

The hologram coat-of-arms shader (`apk_loot/shaders/hologram_fragment.glsl`)
is **not time-driven** — it's gyro-driven. Old time-based code is commented out
and the active path uses only `u_roll`. The CSS replica needs to fake the
gyro tilt with `requestAnimationFrame` + `DeviceOrientationEvent` (or, for
desktop, mouse position).

## §3 Animated Vector Drawables (AVDs)

After filtering library boilerplate, the only AVDs the app brings are the
password-field eye-icon morphs.

### `avd_show_password`

`apk_loot/animations/avd/drawable__avd_show_password.xml`:

```xml
<animated-vector android:drawable="@drawable/$avd_show_password__0">
  <target android:name="eye_mask"      android:animation="@anim/$avd_show_password__1"/>
  <target android:name="strike_through" android:animation="@anim/$avd_show_password__2"/>
</animated-vector>
```

Base vector (`$avd_show_password__0`): an eye outline with a clip-path mask
named `eye_mask` and a diagonal strike-through stroke named `strike_through`.

| Sub-animator | Property | From → To | Duration | Interpolator |
| --- | --- | --- | --- | --- |
| `__1` | `pathData` on `eye_mask` (clip) | path 271 → path 272 | `show_password_duration` = **200 ms** | linear-out-slow-in (`@android:010C000F`) |
| `__2` | `trimPathEnd` on `strike_through` | 1 → 0 | 200 ms | linear-out-slow-in |

Net effect: the strike-through stroke is *withdrawn* (trim end shrinks 100%→0)
while the clip-path mask reveals the open eye.

### `avd_hide_password`

`apk_loot/animations/avd/drawable__avd_hide_password.xml`:

```xml
<animated-vector android:drawable="@drawable/$avd_hide_password__0">
  <target android:name="eye_mask"      android:animation="@anim/$avd_hide_password__1"/>
  <target android:name="strike_through" android:animation="@anim/$avd_hide_password__2"/>
</animated-vector>
```

| Sub-animator | Property | From → To | Duration | Interpolator |
| --- | --- | --- | --- | --- |
| `__1` | `pathData` on `eye_mask` | path 272 → path 271 | `hide_password_duration` = **320 ms** | fast-out-slow-in (`@android:010C000D`) |
| `__2` | `trimPathEnd` on `strike_through` | 0 → 1 | 320 ms | fast-out-slow-in |

The reverse: clip morphs back to the closed-eye shape and the strike-through
stroke is drawn from start→end (trim end grows 0→100%).

### CSS replica recipe

In the replica we can keep both SVG eyes inline and toggle `display`/`opacity`,
or use a single SVG with a CSS-animated `<path>` (cleaner). The drawn-line
effect is the classic `stroke-dasharray` + `stroke-dashoffset` trick.

```css
/* eye icon as SVG, with .strike line + .eye-open / .eye-closed groups */
.pwd-toggle .strike {
  stroke: currentColor;
  stroke-width: 1.8px;
  stroke-linecap: round;
  stroke-dasharray: 24;        /* line is 24 user units long */
  stroke-dashoffset: 24;       /* hidden by default */
  transition: stroke-dashoffset 320ms cubic-bezier(0.4, 0, 0.2, 1);
}
.pwd-toggle[aria-pressed="true"] .strike {
  /* "Password is HIDDEN" — strike fully drawn */
  stroke-dashoffset: 0;
  transition: stroke-dashoffset 320ms cubic-bezier(0.4, 0, 0.2, 1);
}
.pwd-toggle[aria-pressed="false"] .strike {
  /* "Password is VISIBLE" — strike withdrawing */
  stroke-dashoffset: 24;
  transition: stroke-dashoffset 200ms cubic-bezier(0.0, 0, 0.2, 1);
}
/* Eye open/closed clip morph — approximated by swapping path d via CSS variable
   or animating a small group transform. Easiest: a 200/320 ms opacity crossfade
   between two <path> elements. */
.pwd-toggle .eye-open    { opacity: 0; transition: opacity 200ms cubic-bezier(0,0,.2,1); }
.pwd-toggle .eye-closed  { opacity: 1; transition: opacity 320ms cubic-bezier(.4,0,.2,1); }
.pwd-toggle[aria-pressed="false"] .eye-open   { opacity: 1; }
.pwd-toggle[aria-pressed="false"] .eye-closed { opacity: 0; }
```

(`aria-pressed="true"` = "show password" toggle is active → password is hidden
behind dots, eye is closed + strike drawn; `aria-pressed="false"` = clear text,
eye open + strike withdrawn. Match whichever semantic the existing markup uses.)

## §6 Hologram shader timing

Files: `apk_loot/shaders/hologram_vertex.glsl`, `hologram_fragment.glsl`.

The fragment shader declares these uniforms:

```glsl
uniform vec2  u_resolution;
uniform float u_time;
uniform vec2  u_mouse;
uniform float u_globalTime;
uniform float u_pitch;
uniform float u_roll;
uniform sampler2D u_Texture;
```

— but the **active code path does not use `u_time` or `u_globalTime`**.
The entire time-driven simplex-noise hologram is commented out (lines 56-63,
labelled "This is old hologram effect, commented out so if we want to revert
back in the future we can"). What ships in production is a pure roll-driven
alpha multiplier:

```glsl
// active code:
vec4  textureVec = texture2D(u_Texture, v_TexCoord);
float roll       = (abs(u_roll) / 10.0) + 0.2;    // -10..10° → 0.2..1.2 alpha
float r          = textureVec.r * roll;
float g          = textureVec.g * roll;
float b          = textureVec.b * roll;
float alpha      = textureVec.a * roll;
gl_FragColor     = vec4(r, g, b, alpha);
```

**Hardcoded constants worth extracting:**

| Constant | Value | Meaning |
| --- | ---: | --- |
| `abs(u_roll) / 10.0`   | divisor `10.0` | the tilt range that maps to "fully bright" — ±10° of roll |
| `+ 0.2`                | `0.2`          | minimum alpha (the coat-of-arms is never fully invisible) |
| u_roll range (comment) | −60..+60       | what the gyro actually produces |
| u_roll usable range    | −10..+10       | what the designer expected users to tilt |

So in CSS, the hologram is **NOT a CSS @keyframes loop** — it's a one-shot
opacity that you drive from a gyro / pointer position. There's no "shimmer
duration" to match. Use JS to read `DeviceOrientationEvent.gamma` (left/right
tilt = roll) and apply `style.opacity = Math.min(1.2, Math.abs(gamma)/10 + 0.2)`.

For desktop replicas (no gyro), fall back to mouse position over the card —
take the X-offset from card centre, normalise to ±halfWidth, treat as a
roll-equivalent. Optional: also add a *subtle* time-based shimmer (1.5 s
loop) for the "this looks alive" feel even at rest. Recipe in §7.

### Hologram coat-of-arms shimmer

The shipping shader is gyro-driven, not time-driven (§6). For a faithful
replica drive opacity from `DeviceOrientationEvent`; for a fallback give a
gentle 1.5 s loop so the card visibly "lives" at rest.

```css
/* The actual shipped behaviour — JS sets --tilt in 0..1 (clamped) from
   abs(roll)/10. We render exactly the shader's transfer function:
   opacity = clamp(tilt + 0.2, 0.2, 1.2)  ≈  clamp(0.2..1.0). */
.mvr-hologram {
  --tilt: 0;
  opacity: calc(min(1, var(--tilt) + 0.2));
  mix-blend-mode: screen;       /* white on coloured plate, matches APK shader */
  pointer-events: none;
  transition: opacity 80ms linear; /* smooth out gyro jitter */
}

/* Idle fallback shimmer (only when no gyro data has arrived in 1 s) */
@keyframes mvr-hologram-idle-shimmer {
  0%, 100% { opacity: 0.30; }
  50%      { opacity: 0.55; }
}
.mvr-hologram[data-idle="true"] {
  animation: mvr-hologram-idle-shimmer 1500ms ease-in-out infinite;
}
```

```js
// minimal driver
const holo = document.querySelector('.mvr-hologram');
let lastTiltAt = 0;
window.addEventListener('deviceorientation', (e) => {
  const roll = e.gamma ?? 0;           // -90..+90, but typically -10..+10
  const tilt = Math.min(1, Math.abs(roll) / 10);
  holo.style.setProperty('--tilt', tilt.toFixed(3));
  holo.dataset.idle = 'false';
  lastTiltAt = Date.now();
}, { passive: true });
setInterval(() => {
  if (Date.now() - lastTiltAt > 1000) holo.dataset.idle = 'true';
}, 500);
// Desktop fallback: pointermove over the card sets --tilt from X offset.
holo.parentElement.addEventListener('pointermove', (e) => {
  const r = holo.parentElement.getBoundingClientRect();
  const dx = (e.clientX - (r.left + r.width / 2)) / (r.width / 2);
  holo.style.setProperty('--tilt', Math.min(1, Math.abs(dx)).toFixed(3));
  holo.dataset.idle = 'false';
  lastTiltAt = Date.now();
});
```
